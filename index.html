GIT E GITHUB

Primeiramente, antes de tentar entender o que é GitHub, é necessário saber o conceito de controle de versão e Git. Então, vamos lá!

O que é Controle de Versão?

É uma maneira de gerenciar as mudanças ocorridas em um código, ou seja, pode-se dizer que é um sistema que registra alterações em um arquivo ou grupo de arquivos ao longo do tempo, para assim, você se lembrar das versões e recuperar elas. Existem 3 tipos de controle de versão, são eles: Centralizados, Distribuidos e Locais.

→ Centralizados: Desenvolvedores trabalham juntos em um único projeto, tendo apenas um repositório.

→ Distribuidos: Os desenvolvedores do projeto podem trabalhar juntos ao mesmo tempo, com isso, ele é recomendado para projetos que possuem mais de um desenvolvedor onde, todo o repositório é copiado.

→ Locais: Ele armazena todos as alterações na máquina.

O que é GIT?

Git é um controle de versões, desenvolvido por Linus Torvalds. Ele é usado para "controlar as versões". No Git, são armazenados snapshots, ou seja, é como se fosse uma foto da situação do sistema naquela hora. Outra caracterísica importante de ressaltar é que quando uma pessoa faz alguma alteração no trabalho é gerado uma nova versão. Ele registra três estados, são eles: consolidade (commited), modificado(modified) e preparado(staged).

> Consolidade: Ficam armazenados na base local.
> Modificado: O arquivos sofre modificações, porém ainda não teve commit.
> Praparado: Arquivos que irão fazer parte do próximo snapshot.

O git também possui vários comandos, abaixo estão listados alguns.

Git innit: Para inicializar.
Git add: Ele adiciona o conteúdo ao próximo commit.
Git commit: Registra alterações.
Git status: Para saber quais arquivos estão em quais estados.
Git log: Com ele você pode visualizar o que vem sendo feito em uma branch ou avaliar alterações.

O que é Git Hub?

Git Hub é um servidor git remoto, onde você hospeda os arquivos que estão sendo utilizados no Git. Ele trabalha com repositórios(Pastas do Git para poder armazenar).

Qual a importância do GitHub?

A pertir dele, os projetos podem ser desenvolvidos por diversas pessoas de lugares diferentes.

PYTEST

Pyteste é um framwork que serve basicamente para o programador fazer testes, ele deve ser escrito na linguagem Python e o arquivo deve estar com final ".py". Ele é de código-aberto e para fazer a execução, o usuário deverá ir no terminal do aplicativo que está usando e realizar a instalação com o comando pip install pytest e logo depois digitar pytest prosseguido do nome do arquivo. FLASK

Antes de entender o que é Flask, é necessário um breve conhecimento de Framework e Micro-Framework, então vamos lá!

O que é Framwork e Micro-Framework?

Frameworks são códigos prontos com várias funções que serão utilizadas no desenvolvimento de sites. Com isso, ele ajuda você a simplificar a criação de projetos, pois você não precisa iniciar do zero. Segundo o site da lewagon, "ele funciona basicamento como uma peça de quebra-cabeça", já que você vai encaixando ele e conectanto as demais linhas dos códigos. Já os Micro-Framwork são Frameworks "modularizados" mais simples, que permite você executar módulos na ordem correta e lhe ajuda a se organizar.

O que é Flask?

criado por Armin Ronacher, ele é um micro-framwork que usa a linguagem Python para criar aplicações web, ele oferece simplicidade e flexibilidade para o desenvolvedor. Por mais que ele seja um "micro", o Flask pode ser usado tanto em projetos grandes como pequenos.
